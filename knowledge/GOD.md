# GOD

知识库与文档生成任务清单。

## 1. 架构图

读一下这个项目的 README 和顶层目录，给我画一张架构图。

- 前端、后端、数据库、中间件分层画，核心模块写一句话职责。
- 周边基础设施（日志、监控、配置）用一个方框概括即可，不用展开。
- 每一层分为多个小的模块；右边的基础设施也用大框分类；内容使用英文。
- 最后完成后，保存到（路径待补全）。

## 2. 模块图

画一张内部模块依赖图。只画项目自己的模块，外部库不画。

- 若存在循环依赖，用红色标出。
- 将 SVG 图片保存到（路径待补全）。

## 3. 依赖图

综合阅读 README 和项目配置文件，梳理本项目：

- **对外依赖**分为三类：关键依赖、中间件、外部 API。
- 用图表示，每类使用不同颜色。
- 将 SVG 图片生成并保存到（路径待补全）。

## 4. 接口清单

扫描项目中所有接口，整理一份 **REST 接口清单**。

每个接口需包含：

| 项 | 说明 |
| --- | --- |
| 方法 | GET / POST 等 |
| 路径 | URL 路径 |
| 说明 | 一句话描述 |
| 入参 | 主要入参 |
| 返回 | 返回结构概要 |

按模块分组。将 MD 文档保存到（路径待补全）。

## 5. 数据结构

结合项目主要类与数据结构，梳理核心数据模型。

每个模型需列出：

- 字段、类型、一句话说明；
- 标出主键、外键、枚举值；
- 关键模型之间的关系画一张简单的 **ER 图**。

输出文件：

- `/root/hrj/llama.cpp/docs_m/data-model.md`
- `/root/hrj/llama.cpp/docs_m/data-model-er.svg`

## 6. claude.md（Cursor Rule 初稿）

阅读 `doc_m/` 下所有资产，生成一份 Cursor rule 初稿，要求精简，包含：

- 项目定位
- 核心架构
- 关键模块
- 关键约定
- 如何运行
- **禁区**（留空待填）
- **历史包袱**（留空待填）

架构图、接口清单、数据模型的详细内容不要复制进 rule，用链接指向 `doc_m/` 即可。

保存到项目根目录。
